Copilot Tips: Using Up-to-Date Docs in Cursor & Copilot Chat

Goal (10–15 min): Master practical ways to access fresh, trustworthy documentation without leaving your IDE.

Focus on VS Code GitHub Copilot Chat (built-in), Cursor @docs, and optional ref/context tools.

Last Updated: October 29, 2025

GitHub Copilot Chat (VS Code): Built-in Documentation Access

As of October 29, 2025, VS Code Copilot Chat provides powerful built-in ways to access current documentation and code references:

1. Commands for Documentation

  • Generate Docs: Use Inline Chat (Cmd+I or Ctrl+I) on a function or class, or use the sparkle icon (Code Actions) and select "Generate Docs".
  • /explain - Get explanations for selected code with context.
  • /fix - Fix issues with code and explain the solution.
  • /tests - Generate unit tests with documentation.

2. Context Variables (# tags)

  • #file - Reference a specific file. Type # to get file name suggestions
  • #editor - Include the visible code in the active editor
  • #selection - Focus on currently selected code
  • #function - Include the current function or method
  • #codebase - Perform a code search in workspace for relevant context
Example Prompts with Context Variables ▼ 
#file:package.json Can you suggest improvements and cite any best practices docs?

#function Generate JSDoc comments for this function with parameter descriptions

#selection Explain this code and link to relevant MDN documentation

#codebase How do we handle JWT validation? Cite any external docs used.

3. Chat Participants (@ mentions)

  • @vscode - Ask about VS Code features and commands. Example: @vscode How do I enable word wrapping?
  • @workspace - Get context about code in your workspace

4. File Attachments & Drag-and-Drop

  • Use the paperclip icon in Chat to attach files/folders
  • Drag files from Explorer or editor tabs directly into chat
  • Type # in chat input to quickly attach files by name

5. Public Code Matching & Citations

  • When enabled, Copilot may show "View matches" links to publicly available code on GitHub
  • Look for inline citations and links to official docs (MDN, Microsoft Learn, etc.)
  • Click citation links to verify information directly from source

6. Inline Chat & Code Actions

  • Press Ctrl+I (Win/Linux) or Cmd+I (Mac) for inline chat on selected code
  • Use sparkle Code Action (light bulb) to explain code or generate docs directly in editor
  • Ask "Explain this code" or "Update documentation" directly on the line

Note: Citation features and web grounding availability may vary by VS Code version, Copilot license tier, and organizational settings. If citations aren't visible, explicitly ask: "Provide this answer with links to official documentation."

Cursor: Using @docs for Live Documentation

Cursor's @docs command lets you query official documentation directly from chat, fetching, summarizing, and linking sources for quick verification.

  • Targeted lookups: Type @docs followed by a topic, package, or API name. Specify versions when needed for accuracy.
  • Grounding with sources: Ask Cursor to output links and quote relevant passages from the official docs.
  • Side-by-side comparisons: Request comparisons across versions or frameworks with inline citations.
Effective @docs Prompts ▼ 
@docs "react router data routers v6.27" show migration steps from loaders/actions; include links.

@docs "fastapi background tasks" provide minimal example and official reference.

@docs "kubernetes ingress nginx annotations" summarize common settings; link to upstream docs.

@docs "node 22 fs api vs node 20" what changed? cite documentation URLs.

Tip: If @docs returns generic info, narrow your scope by adding version numbers, exact API names, or paste a specific documentation URL and ask for a grounded summary of that page only.

Optional: Context7 and "ref" style tools

Add tool-integrations that fetch official docs and ground answers with citations.

  • Context7 doc lookups: resolve a library and fetch focused docs/examples.
  • Ref-only answers: require quoting from linked sources; reject unsourced claims.
Prompts (tool-integrated chats) ▼ 
#resolve-library-id react
#get-library-docs /vercel/next.js routing - cite sections and link URLs.

Only answer using the fetched docs; if unknown, say so.

Note: These hashtags are examples for tool-enabled chat setups (e.g., MCP-integrated assistants). They are not built-in Copilot Chat commands. Ensure the Context7 tool is configured; otherwise, paste official URLs and ask for grounded summaries.

Live Demo Flow (7–10 min)

Follow this sequence to show students practical documentation access in real time:

  1. Copilot Slash Command: Open a function, type /docs in chat, and watch it generate JSDoc comments. Show how to accept/edit the output.
  2. Context Variables: Select a code block, use #selection Explain this and cite MDN docs. Point out any citation links that appear.
  3. File Attachment: Drag package.json into chat, ask: "Any outdated dependencies? Link to migration guides for major version changes."
  4. @workspace Query: Type @workspace How do we authenticate users? to see codebase-aware answers.
  5. Cursor @docs: Switch to Cursor (if available), query a specific API with version (e.g., @docs "next.js 14 app router"). Show links and quotes.
  6. Verification Step: Click one citation link live and cross-check a claim against the official doc page.

Instructor Tip: If Copilot doesn't show citation links, pivot mid-demo: paste an official URL (e.g., https://react.dev/reference) and ask "Summarize this page and propose changes grounded to the text." This shows the fallback strategy.

Safe Prompting Tips

  • Always verify: Click citation links and cross-check claims against official sources
  • Be specific: Include version numbers, exact API names, and framework details
  • Request sources: Explicitly ask "Provide links to official documentation" if not shown
  • Fallback strategy: If no citations appear, paste an official doc URL and ask for grounded summary
  • Reject unsourced claims: If answer lacks citations for critical info, ask for sources or verify manually

Wrap-up & Key Takeaways

  • VS Code Copilot Chat offers powerful built-in methods for doc access: inline chat, code actions, slash commands, context variables, and citations.
  • Cursor's @docs provides fast, version-specific lookups with source links.
  • Context7 and ref tools add advanced grounding for tool-integrated workflows.
  • Always verify citations and prefer official sources (MDN, Microsoft Learn, vendor docs).
  • Use context variables and file attachments to reduce hallucinations and get precise answers.

Latest Enhancements (Oct 2025)

  • Automatic Participant Detection: Copilot now intelligently routes questions to the best participant (e.g., `@workspace`, `@vscode`) without you needing to specify it.
  • Editor Context Toggle: You now have a setting to control whether your active editor content is automatically included in a chat query.
  • Interactive Symbol Links: Workspace symbols (e.g., function or class names) in chat responses are now interactive links to their definitions.
